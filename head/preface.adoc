== 推薦序
隨著 Linux 的穩定度受人肯定，愈來愈多的商用軟體也採用 Linux 做為基底。
因此在排測硬體相容性時，做為測試基準的作業系統除了 MS-Windows 之外，也必須考慮到 Linux。
而它在 x86 以外的硬體平臺上，更是具有舉足輕重的地位。

擔任硬體相容性測試的同仁，一般都是依循標準的操作流程來執行測試計畫。
雖然並未要求熟悉電路布局或裝置設計，也不必精通晶片控制或程式編寫，
但是多樣的環境與多變的狀況，使得他們需要具備對於電子、機構、韌體以及軟體的通盤理解，
才能憑藉著豐富的經驗，順利辨明各種難題，進而提出可行對策。
可惜由於工作進度的要求，並沒有足夠的時間讓人對於各項技術進行系統性的學習。

作者 William 任職於本公司研發處 Linux 軟體部，負責支援其他部門的相關事務，
包括驗證部的零件承認與相容性測試、硬體製造部的前期測試與出廠驗證，以及客服部的問題復現與對策。
在工作時能夠秉持著不斷學習的精神，積極地藉由書籍、網路與同事的討論，找出最適當的解決方案。
更難能可貴的是，將獲得的知識與經驗記錄下來，避開了個別產品的特定規格之後，仍能從中提取普遍性的整合資訊。

在如今開放原始碼與開放硬體的浪潮之下，自由軟體的價值特徵之一即在於「分享」。
誠然本書的內容並非包羅萬象，也未能精確論證每項細節，
但是簡明扼要的說明，與不時輔以作者的心得經驗，貼切地契合實務上的需求。
因此我十分樂意，將它推薦給 Linux 硬體相容性測試的相關從業人員，
以及所有對這個主題感興趣的電腦使用者。
