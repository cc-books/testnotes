== 推薦序
隨著 Linux 的穩定度受人肯定，愈來愈多的商用軟體也採用 Linux 做為基底系統。
再加上硬體平臺的多樣化，使得市場上非 x86 平臺的詢問度日增。
因此在排測硬體相容性時，做為測試主體的作業系統除了 MS-Windows 之外，
也必須考慮到 Linux 系統。

自由軟體的價值特徵之一即在於「分享」

作者 William 是任職於本公司研發處 Linux 軟體部的工程師，負責支援其他部門的相關事務，
包括驗證部的零件承認與相容性測試、硬體製造部的前期測試與出廠驗證，
以及客服部的問題復現與對策。

擔任硬體相容性測試的同仁，一般都是循著標準的操作流程來執行測試計畫。
雖然並不要求具備裝置設計或修改能力，也不必精通程式的編寫，
但是多樣的環境與多變的狀況，使得他們需要具備對於電子、機構、韌體以及軟體的通盤理解，
才能在豐富經驗的加持下，順利辨識各種難題，甚至提出可行對策。
可惜由於工作進度的壓力，並沒有足夠的時間讓人對於各項技術進行系統性的學習。

誠然本書的內容不可能包羅萬象，也無法深入論證每一項細節，
但是簡明扼要的說明，恰當地切合實務上的需求，還不時輔以作者的心得經驗。
因此我十分樂意，將它推薦給與硬體 Linux 相容性測試相關的從業人員，
以及所有對這個主題感興趣的電腦使用者。
