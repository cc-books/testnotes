== 推薦序

擔任硬體相容性測試的同仁，一般都是循著標準的操作流程來執行測試計畫。
雖然並不要求具備專門的硬體設計或修改能力，也不要求精通軟體程式的編寫，
但是多樣的環境與多變的狀況，使得他們需要具備廣泛的知識與豐富的經驗。

隨著 Linux 的穩定度受人肯定，愈來愈多的商用軟體也採用 Linux 做為基底系統。
再加上硬體平臺的多樣化，使得市場上非 x86 平臺的詢問度日增。
因此在排測硬體相容性時，做為測試主體的作業系統除了 MS-Windows 之外，
也必須考慮到 Linux 系統。

自由軟體的價值特徵之一即在於「分享」

作者 William 是任職於本公司研發處的高級工程師，負責支援 Linux 的相關事務，
包括驗證部門的零件承認與相容性測試、硬體製造部門的前期測試與出廠驗證，
以及客服部門的問題復現與對策。

但是在進度壓力之下，並沒有足夠的時間讓人對於各項概念進行系統性的了解。
