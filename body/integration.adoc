== 整合測試
完成各分項的測試之後，就會進行系統的整合測試，包括效能與壓力測試。

WARNING: 由於整合測試需要長時間運行程式，必須注意系統可用的儲存空間，
是否能夠容納過程中產生的暫存檔案。

=== 效能測試
雖然已有多歀專用的效能測試商用軟體，但是如果受到資源的限制，或是測試項目需求不多，
也可以考慮以下的自由軟體。

==== 中央處理器
文字模式下的 `sysbench` 工具程式，一般會在命令中加入與核心數目相同的執行緒參數，
可以在輸出中提供事件頻率(Events/sec)，以評估處理器計算質數的效能：
====
user$ *sysbench cpu --threads=2 run*
====

==== 記憶體
文字模式下的 `sysbench` 工具程式，預設的命令可以在輸出中提供傳輸速率(MiB/sec)，
做為衡量系統記憶體的寫入效能：
====
user$ *sysbench memory run*
====

=== 壓力測試
由於壓力測試常常搭配溫升、電磁這類變動外部環境的測試，或是更嚴苛的項目（防水、防塵、防震），
因此並不會分開測試個別的元件，而是整體同時進行（包括中央處理器、輸入輸出、記憶體）:
====
user$ *stress -c 4 -i 2 -m 2*
====
TIP: 另一個用法類似的命令 `stress-ng`

此時多半需要搭配系統監控軟體，以即時顯示系統資源的使用情形。
在文字模式下可以執行：
====
user$ *top*
====
或是第三方 `bottom` 套件中的 `btm` 命令，後者是受到前者所啟發的全視窗華麗版本。
如果想要看到圖形顯示則可以使用：
====
user$ *gnome-system-monitor*
====
