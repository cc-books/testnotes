== 套件安裝
安裝完作業系統之後，代表這臺電腦已經具備基本的功能。
如果作業系統的安裝程序沒有順便安裝其他的項目，或是內建的應用軟體無法滿足您的需要，那麼就必須自己想辦法來安裝。

目前流行的各類作業系統當然也考慮到這一點，他們將常用的額外應用軟體以特定的格式封裝起來，
將其包裝成附加的商品或是免費的資源，提供給使用者購買或下載。
每一組封裝起來的軟體稱之為套件，而提供許多套件的集合則稱為套件庫。
由於封裝的格式是公開的規範，因此除了原本作業系統的發行者所製作的官方套件之外，還有一些第三方的愛好者會製作非官方的套件。
同樣地，除了官方套件庫之外，也會有人提供非官方的套件庫。

何謂套件
	msi (Windows)
	rpm (RHEL,Fedora,CentOS,SUSE,...)
	deb (Debian,Ubuntu,Mint,...)

套件構成
	單一壓縮檔案包
	由彼此高度相關的各類檔案組成
	可以設定依存性以串連其他套件

套件製作
	配合程式原始碼編寫規格檔
	在模擬環境中編譯,安裝並打包
	產生兩種:安裝套件,原始碼套件

套件管理
	伺服器端:建立套件庫
	客戶端:將套件庫的網址加入
        注意系統時間是否正確

套件操作
		rpm	deb
	套件庫	dnf	apt-get
	(套件檔	rpm	dpkg)

套件命令
			rpm			deb
	資料更新		--refresh	apt-get update
	套件更新	dnf update		apt-get upgrade
	資料清理	dnf clean all		apt-get clean all
	套件安裝	dnf install 套件名稱	apt-get install 套件名稱
	套件移除	dnf remove 套件名稱	apt-get remove 套件名稱
	字串搜尋	dnf search 字串		apt-cache search 字串
	套件檔安裝	dnf install 套件檔	apt-get install -f 套件檔
			(rpm -Uvh 套件檔)	(dpkg -i 套件檔)
