== 測試準備

=== 需求

客服同事轉發客戶通知時，有時會加上自己的解讀，因此要問清楚哪一部分是客服同事自行補充的資料，最好能要求看到客戶回應的原文內容。

=== 環境
非導體、抗靜電、低磁場、無易燃物

=== 網路
進行測試時所需的資料檔案，一般會放在內部網路的檔案伺服器上，
可能是以 SMB 或是 FTP、HTTP 的協定形式分享出來。

偶爾需要拷貝別臺電腦中的檔案時，通常是以**閃碟**(Flash Stick) 做為交換的媒介，
因為臨時要啟用一個檔案分享服務，或是先將檔案上傳到伺服器再下載，都似乎稍嫌麻煩。

如果需要互相傳輸檔案的機器都位於同一內部網路的話，其實還有更簡便的方法：
使用 netcat 套件中的命令，就可以實現互傳檔案的功能。
先在接收端輸入以下命令：
====
user$ *nc -l > _foo.bar_*
====

此時接收端會進入等待狀態，直到發送端輸入命令：
====
user$ *nc _192.168.1.2_ < _foo.bar_*
====

其中 _192.168.1.2_ 請替換為接收端的網路位址，而 _foo.bar_ 則代表想要傳輸的檔案名稱。
如果命令與輸入資料都正確，卻出現無法連線的錯誤訊息，有可能是被接收端的防火牆所阻擋。
請先在接收端停止防火牆服務，再重新進行前述的操作即可：
====
root# *systemctl stop firewalld*
====
